# Redesign — Phase 2: Shell, Top-Nav, Logo

**Date:** 2026-05-01
**Scope:** Replace the per-screen headers with a single global top-nav (`FFNav`). Introduce the new brand mark and wordmark. Regenerate OS icons. Existing screen content stays intact — only their headers go.

## Decisions (from brainstorming)

- Nav items: **Home / Files / Notebook / Settings / Admin** (Admin only when `user.is_admin`).
- **Notebook** nav item is grayed out + disabled when no tabs are open; clicking it (when active) shows the open notebook stage.
- **Kernels** screen — skipped from nav (handoff item, no existing screen, defer indefinitely).
- **Plugins** — stays as a sub-section in Settings; not promoted to a nav item.
- **TabBar** — kept as-is, placed under FFNav in notebook stage. Three-stack header in notebook view (FFNav → TabBar → notebook TopBar) is acceptable.
- **Search ⌘K** in nav — placeholder input + ⌘K chip; no behavior in Phase 2 (wired in Phase 8 alongside command palette modal).
- **Login** + **Loading** + **KernelPicker** stages — no FFNav. Kernel picker is a full-screen modal so it stays untouched.

## New components

- `frontend/src/components/brand/BrandMark.tsx` — inline SVG, accepts `size` and `className`. Stroke uses `currentColor`; cell rect uses `var(--color-accent)`. From handoff:

- `frontend/src/components/brand/Wordmark.tsx` — `Cell` in `var(--color-text)`, `Forge` in `var(--color-accent)`, Space Grotesk 600, letter-spacing −0.015em.
- `frontend/src/components/layout/FFNav.tsx` — 52px header. Left: BrandMark + Wordmark (clickable → home). Center: nav items with icons (lucide). Right: search input (placeholder, ⌘K chip), avatar pill with dropdown (Settings / Logout).
- `frontend/src/components/layout/AppShell.tsx` — composes FFNav above `children`. Used as wrapper for every authenticated stage except `kernel`.

## Touched files

- `frontend/src/App.tsx` — wraps stages `home/browse/settings/admin/ready` in `<AppShell>`. Adds `/admin` URL routing. Drops `onSettings/onAdmin/onLogout/onBack` callbacks from screen components since FFNav handles those globally.
- `frontend/src/components/HomeDashboard.tsx` — removes its own logout/settings/admin header buttons + welcome banner top section. Keeps content (recent notebooks, quick start).
- `frontend/src/components/Dashboard.tsx` — removes `onBack` and `onSettings` props (and the buttons that use them).
- `frontend/src/components/AdminPanel.tsx` — removes `onBack`.
- `frontend/src/components/Settings.tsx` — removes `onBack`.
- `frontend/src/components/toolbar/TopBar.tsx` — drops the redundant "CellForge" home button at the left of the toolbar (brand now lives in FFNav above it).
- `frontend/src/components/layout/AppLayout.tsx` — drops `onGoHome` prop (FFNav handles home navigation).

## Routing

Existing URL scheme stays. Add only `/admin` push (currently the admin stage doesn't update the URL). Notebook URL `/notebook/<path>?kernel=<name>` unchanged.

## Logo replacement

In-app: `BrandMark` + `Wordmark` consumed by FFNav. No other component currently renders a CellForge logo in v1.1.0 except `LoginPage` (Phase 3 redesigns it; for Phase 2 just swap to the new BrandMark there).

OS-level icons regenerated from a new master SVG:

- `frontend/public/favicon.svg` — replaced with the bracket+cell mark (24×24, currentColor → `#a78bfa`).
- `assets/icon.svg` — rewritten to a 1024×1024 master: pure-black rounded square background, large bracket+cell mark centered, "CellForge" wordmark below in Geist-style fallback (since the SVG can't reliably embed Geist for renderers, fallback to system sans matches old behavior).
- `assets/icon-1024.png` — regenerated via `magick assets/icon.svg -background none assets/icon-1024.png`.
- `assets/icon.png` (256×256) — regenerated by resizing `icon-1024.png`.
- `assets/icon.ico` — regenerated multi-size from `icon-1024.png` (`magick icon-1024.png -define icon:auto-resize=256,128,64,48,32,16 icon.ico`).

AppImage build (`packaging/linux/build-appimage.sh`) copies `assets/icon.png` automatically — no script change needed. Windows build embeds `assets/icon.ico` via `cellforge-app/icon.rc` — no code change needed. Tauri-style icon paths unchanged.

## Verification

1. `npm run build` passes, tsc passes.
2. Dev server: every screen except login shows the FFNav at the top.
3. Nav items navigate correctly; Notebook is grayed when no tabs are open.
4. Avatar dropdown logs out and links to settings.
5. Search ⌘K chip is visible but inert.
6. `assets/icon.png` opens as the new mark.
7. AppImage rebuild (out of scope for this session — verify visually post-implementation).

## Risks

- `HomeDashboard` and `Dashboard` may rely on internal state tied to the old back buttons. Mitigation: only strip the buttons + props, not the screen logic.
- New SVG logo at small sizes (16×16 favicon) may look muddy because the cell rect is small. Mitigation: viewBox + stroke widths from handoff are designed for 24×24, should be fine.
- AppLayout currently uses `onGoHome` to handle dirty-save modal flow when leaving the editor. Need to keep that flow — the dirty-save modal trigger moves up to App.tsx so FFNav clicks on Home/Files etc. while in notebook stage still go through the save check.
